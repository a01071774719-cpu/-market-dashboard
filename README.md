# 시장 동향 (Market Dashboard)

국채 금리 · 지수 · 원자재 · 프리미엄을 **한 화면에서 실시간**으로 보는 개인용 다크테마 대시보드. 카드가 4개 섹션으로 그룹화되어 있고, 모바일에서는 세로로 쌓입니다.

## 섹션 구조 (총 10개 카드)

### 1. 국채 금리
| 항목 | 심볼 | 비고 |
|---|---|---|
| 미국 10년물 국채 금리 | `^TNX` | 값이 곧 금리(%) |
| 미국 2년물 국채 금리 | `2YY=F` | **2-Year Yield Futures**. 야후에 2년물 수익률 인덱스(`^`)가 없어 이 심볼 사용. 값이 곧 금리(%). 단, 야후의 인트라데이 데이터가 거의 없어 **캔들 차트는 6개월/1년 등 일봉 기간에서만** 채워짐(현재가 값은 항상 표시) |

### 2. 지수 (실시간 우선, 인베스팅 → 야후 폴백)
| 항목 | 야후 폴백 심볼 | 선물 |
|---|---|---|
| US 500 | `^GSPC` | `ES=F` |
| US Tech 100 | `^NDX` | `NQ=F` |
| 코스피 | `^KS11` | `^KS11` |

- 각 카드는 **인베스팅닷컴 실시간 값을 서버 사이드에서 우선 시도**하고, 실패 시 **야후로 자동 폴백**합니다. 카드에 실제 출처 배지(`인베스팅` / `야후`)를 표시합니다.
- **현재 인베스팅닷컴은 Cloudflare 봇 차단(HTTP 403)** 때문에 실제로는 **야후로 폴백**됩니다(배지 = `야후`). 아래 [출처 조사](#출처-조사-결과) 참고.
- **지수/선물 토글**은 이 섹션 헤더에만 있습니다.

### 3. 원자재
| 항목 | 심볼 |
|---|---|
| 금 | `GC=F` (USD/oz) |
| 은 | `SI=F` (USD/oz) |
| 비트코인 | `BTC-USD` |

### 4. 프리미엄 (큰 % + 핵심 수치, 캔들 대신 추이 라인)
- **김치프리미엄** (`/api/kimchi`): ((업비트 원화 BTC / 달러원환율) / 바이낸스 USD BTC − 1) × 100. 업비트·바이낸스·야후 `KRW=X` 세 소스를 백엔드에서 호출.
- **금치프리미엄** (`/api/goldkimchi`): (국내 금 g당원화 / 국제 금 g당원화환산 − 1) × 100. 국제 금은 `GC=F`를 환율·그램(1 troy oz = 31.1034768 g)으로 환산. **국내 금은 네이버 금융 국내 금 시세(신한은행 고시 기준)** 사용.
- 플러스면 초록(국내가 비쌈), 마이너스면 빨강. 폴링마다 프리미엄을 인메모리로 누적해 추이 라인 표시.

## 출처 조사 결과

프로젝트에서 확정한 소스별 실제 테스트 결과:

### (1) 미국 2년물 금리 심볼
- `2YY=F` (2-Year Yield Futures) → **3.9%대 값이 금리(%)로 정상 조회됨**. 채택.
- `^FVX`는 **5년물**이므로 사용 안 함. `^UST2YR`·`^US2Y` 등은 `Not Found`. `ZT=F`는 2년 T-Note **가격**(≈103)이라 금리 표시에 부적합.
- 참고: `2YY=F`는 야후 인트라데이 시계열이 거의 없어(1일=0점, 6개월 일봉=123점) 캔들은 일봉 기간에서만 채워짐.

### (2) 인베스팅닷컴 실시간
- `api.investing.com/api/financialdata/{id}/historical/chart/` 및 지수 페이지 → **HTTP 403 "Just a moment..." (Cloudflare 봇 차단)**. 서버에서 우선 시도하되 실패 시 **야후로 폴백**하도록 구현. 현재 배지는 `야후`로 표시됨.

### (3) 네이버 국내 금 시세 (금치프리미엄)  ✅ 연동 성공
- **사용 엔드포인트**:
  `https://m.stock.naver.com/front-api/marketIndex/prices?category=metals&reutersCode=M04020000&page=1&pageSize=10`
- 응답의 `result[0].closePrice`(예: `"191,420"`)를 콤마 제거 후 파싱 → **국내 금 g당 원화 시세**. 이는 네이버 금융의 국내 금 시세(신한은행 고시 기준)입니다.
- `pageSize`는 **10 이상**이어야 함(작으면 `too_small` 에러). `Referer: https://m.stock.naver.com/` 헤더 권장.
- 서버 `/api/goldkimchi`가 이 값을 국제 금 환산값과 비교해 금치프리미엄(%)을 계산합니다. (예: 국내 191,420 vs 국제환산 189,829 → **+0.84%**)
- 네이버 응답 구조가 바뀌거나 막히면 억지 값 대신 **"국내 금 시세(네이버) 연동 실패"**로 표시하고, `domesticError`에 사유를 담습니다.

> 과거 시도(참고): 네이버 `goldDetail.naver`(HTML, 국제금 역산값이라 부적합), KRX `data.krx.co.kr`(gold `bld` 미확인·`LOGOUT`), 한국금거래소(JS 로딩·소매가) 는 모두 부적합해서, 위 네이버 모바일 front-api 로 확정했습니다.

## 백엔드 엔드포인트

| 엔드포인트 | 설명 |
|---|---|
| `/api/chart?symbol=&range=&interval=` | 야후 chart 프록시 (OHLC + 메타) |
| `/api/index?symbol=&range=&interval=` | 지수용: 인베스팅 우선 → 야후 폴백, `source` 포함 |
| `/api/kimchi` | 김치프리미엄 계산 (업비트+바이낸스+환율) |
| `/api/goldkimchi` | 금치프리미엄 계산 (국제금+환율+네이버 국내금) |

## 실행 방법

Node.js 18 이상 필요.

```bash
npm install
npm start
```

브라우저에서 **http://localhost:5173** 접속. 포트 변경: `PORT=3000 npm start` (PowerShell: `$env:PORT=3000; npm start`).

## 기술 스택 · 파일 구조

- 프론트엔드: 순수 HTML/CSS/JS + lightweight-charts 4.2.0 (CDN)
- 백엔드: Node.js + Express (정적 서빙 + 프록시/계산 엔드포인트)

```
├── server.js          # Express 서버 + 프록시/프리미엄 엔드포인트
├── package.json
├── public/
│   ├── index.html
│   ├── styles.css
│   └── app.js         # 섹션/카드 렌더링 + 폴링 로직
└── README.md
```

## 공통 동작 / 참고

- **20초 폴링**으로 전 항목 자동 갱신. 기간 선택(1일~1년)·현재 시각 시계·다크테마 유지.
- **신선도 배지**: 거래소 마지막 체결 시각 기준 90초 이내면 `실시간`, 아니면 `지연`. 미국/한국 정규장 마감 시간대에는 지수·금리가 `지연`으로 표시될 수 있습니다(지수 섹션은 선물 토글로 완화).
- **카드 클릭**: 국채·지수·원자재 → 야후 해당 심볼 페이지, 김치 → 업비트 BTC, 금치 → 네이버 금 시세 페이지. (차트 위 클릭은 페이지 이동 안 함)
- `^TNX`/`2YY=F`는 값이 금리(%)이며 등락폭 단위는 `%p`.
- 개인용 대시보드이며 투자 조언이 아닙니다.
